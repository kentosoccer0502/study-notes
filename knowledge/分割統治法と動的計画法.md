# ✅ 分割統治法（Divide and Conquer）

**考え方：大きい問題を “独立した小問題” に分割 → 個別に解く → 結果を合成して解にする**

### 特徴

- 小問題 **同士が独立**（相互依存なし）
- 再帰で書けることが多い
- 合成（Combine）処理が重要
- 代表例：Merge Sort、Quick Sort、二分探索

### メリット

- 並列化しやすい（独立だから）
- アルゴリズムの構造が単純になりやすい

---

# ✅ 動的計画法（Dynamic Programming）

**考え方：小問題を“全部解いて結果を再利用”し、最適解を導く**

### 特徴

- 小問題 **同士が依存する**
- メモ化 or テーブルで再利用（重複計算の排除）
- 最適化問題に向いている
- 代表例：ナップサック問題、最長共通部分列（LCS）、フィボナッチ（DP 版）

### メリット

- 計算量を劇的に改善（指数 → 多項式）
- 一度計算した結果を使い回すため安定した性能

---

# 🆚 両者の違い（超要点）

| 観点               | 分割統治法             | 動的計画法                 |
| ------------------ | ---------------------- | -------------------------- |
| 小問題の関係       | **独立**               | **依存**（重複あり）       |
| 小問題の結果の扱い | 各回で計算             | **保存して再利用**         |
| 適用する問題       | 構造が分割しやすい問題 | 最適化や重複計算が多い問題 |
| 実装               | 再帰が中心             | メモ化 or 表埋め           |
| 例                 | マージソート           | ナップサック、LCS          |
